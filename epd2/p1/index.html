Objetivos

  • Aprender el uso del entorno de desarrollo NetBeans IDE 4.1 y familiarizarse con las acciones básicas del mismo.
  • Edición, compilación, ejecución y depuración de programas.

Conceptos

NetBeans IDE 4.1 es un entorno de desarrollo para la creación de aplicaciones en lenguaje Java desarrollado por Sun Microsystems Inc. y distribuido de forma libre. Éste será el entorno de desarrollo que se empleará durante el desarrollo de las sesiones prácticas de esta asignatura.

NetBeans ha sido desarrollado empleando Java y por lo tanto que se ejecuta sobre la Máquina Virtual Java. Debido a esto, para poder usar NetBeans deberá disponer de dicha máquina virtual, así como las herramientas de desarrollo de Java, todo ello incluido en J2SE JDK, el kit de desarrollo Java 2 en su versión estándar. La versión 4.1 de NetBeans requiere para su correcto funcionamiento que la versión 5.0 o superior de J2SE JDK esté instalada previamente.

Para instalar NetBeans 4.1 en su ordenador personal deberá descargar y ejecutar los siguientes instaladores siguiendo el orden indicado:

  1. J2SE JDK 5.0 Update 8 (o versión superior): http://java.sun.com/javase/downloads/index.jsp
  2. NetBeans 4.1: http://www.netbeans.org/community/releases/41/

Recientemente, Sun Microsystems está ofreciendo a los usuarios la versión 5.0 de NetBeans. Podrá descargar este programa, junto con la versión más reciente de J2SE JDK, todo en un único programa de instalación, en la siguiente dirección:

http://java.sun.com/j2se/1.5.0/download-netbeans.html

1. Pasos en la creación de un programa

Edición
La edición no es más que escribir el código del programa propiamente dicho. Dado que Java es un lenguaje totalmente orientado a objetos, todo código (incluido el programa principal) debe estar encapsulado en una clase. Una vez escrito el código en la clase correspondiente, ésta debe ser almacenada en un fichero cuyo nombre coincide exactamente con el nombre de la clase añadiendo la extensión ".java".
Compilación
La compilación es una fase automática en la que el compilador de java (javac) transforma el código fuente en código máquina (bytecode) interpretable por la máquina virtual, creando un fichero de igual nombre que el “.java”, pero con extensión “.class”. En caso de que el código Java contenga errores (sintácticos o semánticos), el compilador no produce el fichero “.class”, y avisa al programador de los errores existentes, su tipo y localización en el código mediante un mensaje en pantalla. Estos errores se denominan errores de compilación.
Ejecución
Una vez compilado el programa, la máquina virtual interpreta el código máquina y lo ejecuta en la plataforma concreta donde está instalada (sistema operativo + arquitectura hardware). En esta fase también se pueden producir errores por mal funcionamiento del programa, denominados errores de ejecución, que provienen de una confusión del programador al haber escrito un programa que no realiza las operaciones que deseaba. Estos errores son más difíciles de corregir, ya que el compilador no indica qué error hay ni dónde se produce, ni es posible que lo indique ya que para él el programa está escrito correctamente, siendo necesario en la mayoría de los casos depurar el programa para encontrar la parte del mismo que no hace lo que deseamos.
Depuración.
La mayoría de los entornos de programación permiten depurar programas realizando una ejecución paso a paso, es decir línea a línea del programa, y consultar los valores que van tomando las variables y objetos, de manera que podemos hacer una traza para detectar dónde y por qué se producen los errores y corregir los mismos.

2. El entorno NetBeans IDE 4.1

El entorno de desarrollo NetBeans IDE 4.1 integra, entre otras, las siguientes herramientas:

  • Editor orientado la creación de código Java: Este editor resaltará las palabras clave, expresiones, constantes, etc.
  • Compilador de código fuente Java.
  • Depurador visual (Debuger), que permite visualizar y modificar el contenido de variables.
  • Visor de proyectos y ficheros que permite visualizar la jerarquía de las clases utilizadas en los programas así como los diferentes ficheros creados y generados durante la edición, compilación y ejecución de programas.
  • Herramientas complementarias para la creación de aplicaciones complejas.

Como se muestra en la Figura 2, NetBeans presenta el aspecto de una aplicación común Windows, con la barra de menús, la barra de botones y varios paneles de acción. El panel principal es el de edición, donde se escribe el código java de las diferentas clases. A la izquierda encontramos los paneles de proyecto y navegador, con los que podremos acceder a cada una de las partes (clases y librerías) del proyecto abierto en cada momento. Por último, en la parte inferior tenemos el panel de salida, en el que se presenta toda la información sobre la compilación, errores y ejecución del programa.

3. Creación de un proyecto

Normalmente, una aplicación no está compuesta por una única clase, sino que contiene varias clases que interactúan entre sí, cada una de las cuales resuelve parte del problema principal de la aplicación. Un proyecto no es más que una unidad lógica que agrupa todos los componentes (clases, librerías, etc.) que integran la aplicación Java. Por ello, cada vez que queramos implementar una nueva aplicación, crearemos un proyecto donde agruparemos las clases que resuelven el problema que nos ocupa.

Para crear un proyecto simple (aplicación Java con interfaz de texto) se siguen los siguientes pasos:

  1. Elegimos la opción New Project del menú File. Aparece el cuadro de diálogo de la Figura 3.
  2. Seleccionamos Categories: General y Projects: Java Application. Pulsamos el botón Next.
  3. En el cuadro de diálogo de la Figura 4, damos nombre al proyecto (Project Name) y seleccionamos la localización donde se almacenarán los ficheros correspondientes al nuevo proyecto. Recuerde la localización para poder copiar su trabajo y conservarlo. Todo lo que se guarda en los ordenadores de las aulas se borrará al finalizar el día.
  4. En este mismo cuadro podemos seleccionar establecer el proyecto como principal (Set as Main Project) y crear y renombrar de manera automática la clase principal que contendrá el método main (Create Main Class).
  5. Una vez seleccionadas las opciones oportunas, pulsamos el botón finish. Con ello, se crea el proyecto junto a todos los ficheros necesarios para su compilación y ejecución. Ya podemos insertar el código java del programa.

4. Edición del código

Una vez creado el proyecto, podemos añadir clases y código a estas clases. Para poder ejecutar la aplicación, es necesario que exista una y sólo una clase principal que contenga el método main. Si durante la creación del proyecto no se seleccionó “Create Main Class ”, será necesario añadir una. Para ello, se selecciona la opción New File del menú File y posteriormente se elige Java Main Class (si la clase es principal) o Java Class (si la clase no es principal). Posteriormente se le da nombre a la clase en el cuadro de diálogo y se pulsa finish. La clase es añadida al proyecto de forma automática, creándose el fichero con extensión .java correspondiente.

Una vez que tenemos la clase principal, podemos acceder a ella mediante el panel de proyecto y añadir código en el panel de edición. Por ejemplo: 

public class Principal {
   public static void main(String[] args) {
      System.out.println("Hola Mundo");
   }
}

NetBeans aplica al mismo tiempo que estamos editando el código, un análisis sintáctico de los que estamos escribiendo, de manera que detecta errores sintácticos en tiempo de edición. Cuando se produce un error de este tipo, aparece un pequeño cuadro rojo junto a la línea donde se produce el error. Si situamos el ratón sobre uno de estos cuadros, nos indica cual es el error que hay en la línea. Es importante saber que el programa no puede ser compilado hasta que todos estos errores son corregidos.

5. Compilación y Ejecución

Cuando el programa ha sido escrito, y después de corregir los fallos sintácticos que marca el editor, nos disponemos a compilarlo y ejecutarlo. Podemos compilar todos los ficheros de código Java (con extensión de archivo .java), uno a uno mediante la opción compile ”fichero.java” del menú Build. Sin embargo, lo más común es dejar que el entorno compile y posteriormente ejecute el programa usando la opción Run Main Project del menú Run. Esta acción compila automáticamente todos los ficheros que no habían sido previamente compilados y ejecuta el programa resultante. Toda la información sobre la compilación, así como la salida del programa, se muestra en el panel de salida. Si durante la compilación se han producido errores, el programa no llega a ejecutarse, mostrándose dichos errores en el panel de salida. En este caso, podemos acceder directamente a la línea en la que se ha detectado error simplemente pinchando sobre el mensaje de error correspondiente.

6. Depuración

Como se ha comentado anteriormente, un programa no puede ser ejecutado hasta que todos los errores de compilación han sido corregidos, aunque esto no garantiza el buen funcionamiento del programa. Cuando un programa funciona (se ejecuta) pero no realiza su cometido de manera correcta debido a un error por parte del programador, decimos que se producen errores de ejecución. Estos errores no son detectados por el compilador, por lo que es necesario ejecutar el programa paso a paso para realizar una traza del mismo para localizar el error. A esta fase se le denomina depuración (debug). NetBeans incorporan herramientas de depuración a las que podemos acceder mediante el menú Run.


Figura 5. Opciones de Depuración

Para arrancar el modo depuración, seleccionamos la opción Debug Main Project, mientras que para detener la ejecución elegimos Finish Debugger Sesion.

Breakpoints (Puntos de Ruptura)
Sirven para suspender temporalmente la ejecución del programa en una línea de código cuando se arranca mediante la opción Debug Main Project. Así, podemos examinar el valor de las variables o continuar la ejecución del programa paso a paso. Los puntos de ruptura se insertan con la tecla Ctrl+F8 o bien con la opción Toggle Breakpoint.
Ejecución paso a paso (Opciones de Avance)
Una vez suspendido el programa en un punto de interrupción, podemos ejecutar el programa línea a línea, para comprobar su correcto funcionamiento. Las opciones de avance paso a paso son:
  • Step Over (F8): ejecuta la siguiente línea de código. Si la siguiente línea es una función, la ejecuta sin ejecutar paso a paso el código de dicha función.
  • Step Into (F7): ejecuta la siguiente línea de código. Si esa línea es una llamada a otra función, el programa entrará en el código de esa función para ejecutarla paso a paso.
  • Step Out (crtl+F7): ejecuta sin parar en cada línea el código restante hasta salir del método cuyo código se está ejecutando actualmente.
  • Run to Cursor (F4): continúa la ejecución del programa, sin parar en cada línea, hasta llegar a la línea de código en la que situemos el cursor. Esta opción es equivalente a crear un punto de ruptura en dicha línea.
Cuando empezamos a ejecutar un programa en modo depuración, aparecen una serie de nuevos botones en la barra de herramientas con estas mismas opciones. Las diferentes opciones de depuración también pueden ser invocadas pulsando las teclas de función, lo cuál suele ser un a práctica habitual, por lo que es muy recomendable que se familiarice cuanto antes con la tecla que corresponde a cada opción.
Ventanas de Depuración
Para ver los valores de las variables, podemos utilizar las ventanas de depuración. Algunas de estas ventanas se abren directamente cuando iniciamos la ejecución. Si esto no ocurre, podemos abrirlas desde el menú Windows, opción Debugging.
  • Ventana de Observación (Watches): muestra el valor de aquellas variables que seleccionemos con la opción New Watch.
  • Ventana de Variables (Local Variables): muestra el valor de las variables definidas dentro de la función actual.
  • Pila de llamadas (Call Stack): Muestra la pila de llamadas, es decir los que están métodos en ejecución en ese momento.

public class Experimento3 {

      int i = 10;
      int j = 4;
      double k;

  1. ¿Qué valor debería tomar la variable k?

E4 El siguiente programa pretende determinar cual es la variable que almacena el valor máximo de las tres que se definen. Edítelo y ejecútelo. 

public class Experimento4 {
   public static void main(String[] args) {
      int x, y, z;
      x=10;
      y=x/2;
      z=x+y/2;
      if(x>y)
         if(x>z)
            System.out.println("x es el máximo");
         else
            System.out.println("z es el máximo");
      else
         System.out.println("y es el máximo");
   }
}
  1. ¿Qué valores toman cada una de las variables?
  2. ¿Funciona correctamente? Ejecute el programa en modo depuración y analice dónde está el problema.
  3. Corrija el programa, vuelva a compilarlo y ejecutarlo.
